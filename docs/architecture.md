# AimiliVPN 架构与技术实现说明

本项目是一个**以 Xray 为核心**的多协议代理管理系统，辅以 VPNGate (OpenVPN) 动态节点池作为其出站方案之一。

---

## 📂 目录结构与模块划分

* **/ (根目录)**: 仅保留安装脚本 `install.sh`、配置文件及文档。
* **/backend**: 核心后端控制台。
  * `vpngate_manager.py`: 程序启动入口。
  * `app/`: 模块化后端代码。
    * `main.py`: 主进程生命周期与网关初始化。
    * `core/xray.py`: **系统核心**，管理 Xray 的账户、入站协议（VLESS 等）、出站路由、订阅生成及进程控制。
    * `core/vpn.py`: **出站插件之一**，管理 VPNGate 节点的抓取、测速与 OpenVPN 隧道的建立。
    * `api/`: 提供轻量级的 Web API 服务。
* **/web**: 网页管理面板前端静态资源。
* **/proxy**: 本地代理转换网关，桥接容器/系统网卡与 HTTP/SOCKS5 代理。
* **/utils**: 通用工具集。

---

## ⚙️ 核心技术实现原理

### 1. 系统核心：Xray 分发与路由 (Xray Core)
* **协议入站**: 支持 VLESS 等主流协议，为客户端连接提供高安全性的接入入口。
* **账户与订阅管理**: 动态维护用户 UUID 凭证，自动生成节点分享链接，并提供符合标准格式的 Base64 订阅分发。
* **出站路由分配**: 
  * 支持配置多个出站节点（包括用户自建订阅链接、Cloudflare WARP 节点、动态 VPNGate 代理）。
  * 自动解析路由规则（如绕过大陆、拦截广告等），对客户端请求进行分流分发。

### 2. 动态出站：VPNGate 节点池管理 (OpenVPN Outbound)
* **节点自动拉取**: 守护线程定期从 VPNGate 接口及镜像源拉取节点列表。
* **并发筛选**: 自动对节点进行多线程连接性与延迟测试，将可用节点加入本地节点池。
* **隔离隧道连接**: 建立 OpenVPN 隧道网卡（`tun0`），并配置独立的 Linux 路由表（Table 100），确保 VPN 的默认路由修改仅作用于特定的代理出口，不影响宿主机主网络。

### 3. 网关桥接与分流 (Proxy Bridge)
* **本地网关**: 监听 `7928` 端口，将代理流量桥接至 OpenVPN 隧道网卡 `tun0`。
* **Xray 出站路由**: 当路由规则决定将流量送往 VPNGate 时，Xray 会将流量转发至本地网关端口 `7928`，完成“客户端连接 -> Xray 路由分流 -> 代理网关 -> VPN 隧道出站”的完整链路。
* 
### 4. 统一 Web 面板 (Dashboard)
* 后端通过原生 HTTP API 统一对外提供统计数据，展示 CPU/内存、流量统计、用户流量消耗情况及活动出站状态。
